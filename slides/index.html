Traditional Servers

  • Allow you to choose the server (size, memory, disk space etc.)
  • Pay for the amount of hours it's running (even if not being used)
  • AWS EC2, DigitalOcean, Heroku
  • Can be scaled up in case of higher requests

Enter Serverless üòé

  • A way to run your code without manually provisioning servers
  • Your main focus is shipping code and not server management

AWS Lambda

“AWS Lambda is a serverless compute service that lets you run code without provisioning or managing servers, creating workload-aware cluster scaling logic, maintaining event integrations, or managing runtimes. With Lambda, you can run code for virtually any type of application or backend service - all with zero administration”

Comparisions with our old friend

  • Deployment is easier
  • Scaling doesn't need effort
  • Automated security
  • Easy to set an execution and usage limit
  • Support for all famous runtimes (Node, Python, Go etc.)
  • Ability to run containerized applications (Docker images)
